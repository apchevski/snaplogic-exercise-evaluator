- `/grade` now writes a **short, general, descriptive-only `## Overall` paragraph** instead of an exhaustive recap. Three hard limits, enforced in both full and single-task mode: **(1) short** — 1–2 sentences leading with the headline (pass/fail count + total points), then overall performance / recurring themes in general terms; **(2) general** — no mention or enumeration of specific tasks/slugs or their individual issues (those live in the per-task sections; recurring best-practice *categories* stated generally are fine); **(3) no recommendations** — no suggestions for improvement, next steps, "the single highest-value area to improve", or "fixing X would make it flawless" advice (the Overall describes how the submission did, it never advises). The ambiguous-fuzzy-name-match note goes in that task's per-task section rather than Overall. Documented in new convention `.claude/conventions/grade-overall-short.md`; SKILL.md (both modes) and the `cmd_report` Overall placeholder in `evaluator/grade.py` updated to match; per-task sections still carry all the detail.
- **Output matching is now column-order-insensitive** for every exercise. The `output_match` hard gate previously compared the header as an ordered list, so a student who emitted the right columns in a different order (e.g. `Department ID, Employee ID, Name` instead of `Employee ID, Name, Department ID`) failed even when the data was correct. Now the gate compares the **column-name set** (same names, same count, any order); when the order differs it **realigns the student's rows to the solution's column order by name** before the row-multiset comparison, so a pure reordering passes. Only a missing column, an extra column, a different column count, or genuinely different row data fails. `columns_only` mode likewise compares the column-name set in any order. Row order was already ignored; now column order is too. (Implemented in `evaluator/hard_gates.py` via `_columns_match` / `_build_column_index_map` / `_reorder_row`; rules text, README, and architecture notes updated.)
- Fix: the **exact-match output gate now reads `.xlsx`**, not just CSV. `check_output_files_match` (the default `columns_only=False` path) called `_read_csv` directly on both files, so an XLSX output crashed the whole `plan` run with `UnicodeDecodeError` on the binary. Added `_read_tabular` (sniffs the `PK\x03\x04` zip magic exactly like the header-only path already did) and `_read_xlsx_rows` (places cells by their `r="B2"` column reference so sparse rows stay aligned, pads body rows to header width). The header-only/`columns_only` path was already XLSX-aware; only the row-comparison path was not.
- Fix: the `evaluator.grade` CLI now forces **UTF-8 on stdout/stderr** (`errors="replace"`) at the start of `main()`. Gate details and pipeline names carry non-ASCII text (en-dashes in task names, accented characters in sample output rows); on a default Windows cp1252 console a single `print` of such text raised `UnicodeEncodeError` and aborted an otherwise-healthy run mid-progress.
- Fix: `exercises/task_04_born_on_friday/task.json` was missing `"output_match_mode": "columns_only"`, so the random-user API exercise (whose rows differ every run) was graded with exact row matching and produced a false FAIL on correct submissions. Added the flag; the columns-only gate compares only the header set (format-aware for `.xlsx`) as the task notes already described.
- Fix: `/prep` sync now **preserves a hand-set `output_match_mode`** when it regenerates a single-output `file_writer` task.json. `_write_file_writer_task_json` rebuilt the dict from only slug/path/filename, so reconciling any drift (e.g. a pipeline rename) on `task_04_born_on_friday` would silently drop `"columns_only"` and revert the gate to exact matching — reintroducing the false FAIL (and, for `.xlsx`, the crash). The writer now carries forward a non-default `output_match_mode` from the existing task.json; fresh creates still omit it (default `exact`). The multi-output and `triggered_task` paths already preserved it via `_rewrite_solution_pipeline_path`.
- Add `exercises/task_05_multiple_flows_one_pipeline/notes.md` — task-specific AI guidance for the "Multiple Flows – One pipeline" exercise. Whitelists two correct structural variants as PASS with no deduction: (1) adding the record index via a **Sequence** snap (solution) **or** a **Mapper** using `snap.in.totalCount`, and (2) the solution's **Filter + single Mapper before a single Router** **or** a single Router followed by **3 identical per-branch Mapper snaps** — explicitly overriding the universal "No extra Mapper snaps" (`-1`) rule for those 3 Mappers. All other universal rules still apply unchanged.
- **Rename the `csv_writer` task type to `file_writer`** (format-neutral). Along with it: task.json keys `output_csv_filename` / `output_csv_filenames` → `output_filename` / `output_filenames`; the `prep sync --output-csv` flag → `--output-file`; and the internal gate names `csv_output_match` / `csv_output_present` → `output_match` / `output_present` (plus the helpers/vars behind them). The old name implied CSV-only, but the output can be any tabular format the comparison gate handles — e.g. `task_04_born_on_friday` writes `.xlsx`. **Back-compat:** the old `task_type` value, the old task.json keys, and the `--output-csv` flag are all still accepted and normalize to the new names; the repo's own task.json files were migrated. Evaluator code, README, and both SKILL.md files updated.
- Add **multi-output `file_writer`** support: a task.json may now list several output files under `output_filenames` (an array) instead of the single `output_filename`. The pipeline must reproduce **every** file exactly to PASS — the `output_match` hard gate compares all of them and aggregates per-file PASS/FAIL into one result (its name is unchanged, so AI partial-credit routing is identical). A single-output task.json keeps working unchanged (it's just a length-1 list internally); use exactly one of the two keys. `/prep` can't auto-decide whether multiple writers mean "one canonical output" vs "all are deliverables", so multi-output task.json is hand-authored like `triggered_task` — sync then fetches every `expected/<file>` and prunes the rest, and preserves the hand-authored filename list (only fixing a drifted pipeline path). MISSING now requires **all** output files to be absent in SLDB (student never ran the pipeline); a partial run fails the output gate for AI partial credit instead. Added for `task_05_multiple_flows_one_pipeline` (3 reports: `Trng_FilterEven.csv`, `Trng_IDbelow15.csv`, `Trng_IDabove15.csv`). README, `/prep`, and `/grade` SKILL.md updated.
