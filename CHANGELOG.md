# Changelog

## [Unreleased]

- Broaden the project `.claude/settings.json` allowlist for `/grade`: replace the four narrow `evaluator.grade plan|report` / `evaluator.prep survey|sync` and `python.exe -c` rules with `Bash(.venv/Scripts/python.exe:*)` plus a `Bash(.venv\Scripts\python.exe:*)` backslash variant, and add `Bash(python:*)` / `Bash(python.exe:*)` for PATH-resolved invocations. Rationale: ad-hoc `python -c` inspection scripts emitted during AI judgment were prompting for approval — the existing `-c:*` rule wasn't matching on Windows because the bash matcher saw `.venv\Scripts\python.exe` (backslashes) while the rule used forward slashes. Goal is for `/grade` to run end-to-end without permission prompts.
- Per-student **Total** denominator is now always `(total exercises) × 10`, not `(graded exercises) × 10`. MISSING / NEEDS_PREP entries still aren't AI-judged (their per-task row shows `Points: —/10`), but they now contribute `0` to the numerator and `10` to the denominator instead of being dropped from the math. Rationale: a student who skipped half the exercises should see that reflected in their total — previously, a submission with 2/7 attempted and both failing showed `8/20 (across 2 graded exercises)`, which read as "did well" when really they did 8/70. Updated: `evaluator/grade.py:cmd_report`, `.claude/conventions/grade-points-system.md`, `README.md`.
- Reclassify `csv_output_present` (HTTP 404 on the student's expected output CSV) **and** `triggered_task_exists` (no Triggered Task with the convention name) from procedural **FAIL** (0/10) to **MISSING** (not graded; excluded from totals). Rationale: a submission with no runnable deliverable can't really be graded — for csv_writer that's "no output uploaded," for triggered_task that's "no Task to invoke." Treating both as MISSING keeps the per-student average meaningful for the exercises the student actually attempted. New exit code `4` from the evaluator signals this; `evaluator/grade.py:cmd_plan` reads it and writes `status: "missing"` into the manifest. Updated: `evaluator/evaluate.py` (new `_write_missing_artifact`), `evaluator/grade.py`, `exercises/general_evaluation_rules.md`, `.claude/conventions/grade-points-system.md`, `.claude/skills/grade/SKILL.md`, `README.md`.
- Soften the bonus-question placement rule. Previously: answer in a sticky note / `info.notes` / anywhere but the version notes → `-1`. Now: a correct answer in any field is mentioned (Notes, no deduction); only a missing-or-wrong answer costs `-2`. Rationale: don't punish a correct answer for being filed in the wrong field; tell the student the canonical location, but don't deduct.
- Add universal best-practice rule: **zip inputs must use a ZipFile Read snap (`-5 points`)**. If the task's input is a zip archive and the student reads an already-extracted CSV directly (i.e., they unzipped manually outside the pipeline), deduct `-5`. Promoted from a task-specific `-2` rule in Task 01's `notes.md`; the task notes now defer to the universal rule. This is the first deduction worth more than 2 points — it reflects skipping a core task requirement rather than a best-practice slip. The SKILL.md `points_deducted` guidance and convention file are updated to acknowledge values beyond `0`/`1`/`2`.
- Clarify the floor-at-0 rule for PASS: even if soft-rule deductions sum past 10, the **verdict stays PASS** (the output matches the solution); points just clamp to 0. Verdict and points are two separate signals — verdict comes from hard gates, points from rule deductions.
- Split FAIL into two flavors so a near-correct pipeline isn't ranked the same as an empty one. **Output-mismatch FAILs** (`csv_output_match`, `triggered_task_responses_match`) now route to the AI judge for partial credit — the AI applies the same rule-based deductions it uses for PASS and emits `verdict: "fail"` with `points = max(0, 10 − Σ deductions)`. **Procedural FAILs** (pipeline name mismatch, no Triggered Task with convention name, no output uploaded to SLDB) remain at 0 points with no AI invocation. Renamed the synthesized 404 gate from `csv_output_match` → `csv_output_present` so the two failure modes are routable separately. `evaluator/evaluate.py` gained `_OUTPUT_MISMATCH_GATES` + `_handle_gate_failure`; `.claude/skills/grade/SKILL.md` now tells the AI to read `hard_gates` and decide the verdict (pass vs fail) before deducting; `general_evaluation_rules.md` + `.claude/conventions/grade-points-system.md` updated; README updated.
- Replace the three-verdict scale (`pass` / `pass_with_minor_issues` / `fail`) with **PASS / FAIL / MISSING + 0–10 points**. PASS starts at 10 and the AI deducts using point values written into `exercises/general_evaluation_rules.md` and per-exercise `notes.md` (each soft rule now states `-2 points`, `-1 point`, or *mention only*). FAIL = 0 points (hard-gate failure, AI not invoked); MISSING = `—` (no matching pipeline; excluded from totals). The AI can only ever emit `pass` — escalation to `fail` is reserved for hard gates, and `missing` is set by the planner. Issues with no governing rule are surfaced under **Notes** with 0 deduction. Report header now shows a per-student `Total: X/Y` across graded exercises. Updated: `exercises/general_evaluation_rules.md`, every `exercises/*/notes.md`, `evaluator/grade.py` (renderer + buckets), `evaluator/evaluate.py` (FAIL emits `points: 0`), `.claude/skills/grade/SKILL.md` (new `evaluation.json` schema with `points` + per-difference `points_deducted` + `rule_source`), and new convention `.claude/conventions/grade-points-system.md`.

Per-task notes may still override any universal rule when the exercise calls for it.
