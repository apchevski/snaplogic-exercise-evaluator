# Changelog

## [Unreleased]

- **Every grading run now refreshes the AI Overall summary — scoped runs included.** A single-exercise **Regrade** or a subset selection previously merged its results into the stored report but left the Overall paragraph (and the dashboard summary) stale; now every run ends with the small Overall judge call over the merged report. A multi-exercise scoped job makes that call once, after its last exercise (`run_grade(refresh_overall=...)`). Two side effects: a scoped run on a never-graded student now gets an `## Overall` section inserted into its minimal report (previously the refresh — and a human's inline Overall edit via `_replace_overall_in_md` — silently no-opped without the section), and a fresh AI rewrite clears the paragraph's `overall_summary_edited_by/at` provenance, so the UI correctly shows it as AI-written again.

- **The judge-model default now comes from deployment config everywhere — no constant can shadow a user's choice.** Audit result of "the user's picked model must always win": the per-user flow already worked end-to-end (Settings → `judge_model` on the USER row → job's `requested_by` → `apply_user_overrides` sets `JUDGE_MODEL` → every `AIJudge` call path uses it, sync, batch, and Overall-summary alike). One gap fixed: `GET /v1/settings` reported `default_model` from the Python constant, while the worker's real fallback is the Terraform-set `JUDGE_MODEL` env — which the API Lambda didn't even receive. Now the api-gateway module passes `JUDGE_MODEL` too (single `judge_model` variable feeds both Lambdas; **needs `terraform apply`**), and a new `shared_judge_model()` helper reports the deployment value — reading the pre-override base env so a warm container still carrying another requester's model can't leak into the picker (regression test added). The code constant is only the last-resort fallback.

- **The default judge model is now Claude Sonnet 5** (was Sonnet 4.6 — same price, better judgment; supersedes the June "locked" Sonnet 4.6 decision, recorded in `.claude/architecture.md`). Applies to `DEFAULT_JUDGE_MODEL` in `evaluator/ai_judge.py`, the `judge_model` Terraform defaults (production env + sqs-worker module — takes effect in AWS on the next `terraform apply`), and `.env.example`. Gradings by users with no personal model stored — and existing rows with a null `judge_model` — now run on Sonnet 5; a stored explicit model is untouched.

- **The full "Grade all exercises" run now judges via the Anthropic Message Batches API (~50% cheaper AI cost).** Clicking **Grade** with every exercise selected (the default) submits all of a student's AI-judging calls as one batch billed at half the standard token rate; a **subset** selection (one or several exercises checked) and the per-card **Regrade** stay on the instant **synchronous** path. Because the Batch API is asynchronous, a full run is split across two worker invocations — *submit* (deterministic hard gates → fire the batch → stash the plan scratch in S3) and *collect* (a delayed, self-enqueued SQS message polls the batch; once it ends, per-exercise results are written, the report and AI overall summary are rendered, and the student row refreshes). Results are therefore **not instant** — usually a few minutes, occasionally up to an hour — and the grade dialog now explains this in plain language before you confirm; the student row shows a **Batch grading…** status and updates automatically when the batch finishes (you can close the dialog and leave the page). Backend: `evaluator/ai_judge.py` gains batch build/submit/collect that reuse the exact synchronous request params (so the prompt-cached rules prefix still gives cache reads) plus a 50% discount in the cost estimate; `evaluator/runner.py` gains `submit_grade_batch` / `collect_grade_batch`; `evaluator/store.py` persists the scratch tree to S3 across the async gap; `backend/src/worker.py` runs the two-phase job, holds the per-student lock across the wait (refreshing its 30-min TTL on each poll), and re-enqueues a bounded delayed poll (safe to retry — re-reading a finished batch is free, so a transient collect error doesn't dead-letter a grade whose paid batch already succeeded). **Deploy note:** needs `terraform apply` — the worker Lambda gains `sqs:SendMessage` on its own queue (the self-redrive poll), a scoped `s3:DeleteObject` on `jobs/*` (scratch cleanup), and a `QUEUE_URL` env var. Deploy backend + frontend + infra together.

- **The report editor now covers a task's whole evaluation, not just its summary.** The pencil on a task card previously opened a single textarea for the AI's summary text. It now opens an inline editor (`frontend/src/components/TaskEvaluationEditor.tsx`) for the entire evaluation: the summary, the **deductions & notes** list — add, edit, or remove each item's area, description, −points, rule source, and reasoning — and the **bonus-question** assessment.
